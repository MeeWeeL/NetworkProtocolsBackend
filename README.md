# Network Protocols Backend

Backend-сервис экспериментального стенда для выполнения единых сценариев через REST, SOAP, GraphQL, WebSocket и gRPC.

## Назначение

Сервис содержит общее доменное ядро сценариев `S1-S9`. Транспортные адаптеры обращаются к одной и той же прикладной логике, поэтому различие в измерениях связано с протоколом и режимом соединения, а не с разными бизнес-операциями.

## Возможности

- Единая реализация сценариев чтения, записи, потоков событий и длительной сессии.
- REST, SOAP, GraphQL HTTP, GraphQL WebSocket, WebSocket и gRPC endpoints.
- Возврат серверного времени обработки, размера полезной нагрузки, контрольной суммы и служебных метаданных.
- Поддержка параметров серии: размер полезной нагрузки, число событий, управляемая задержка, профиль нагрузки и режим ошибки.

## Структура проекта

- `src/main/kotlin/.../service` - доменная логика сценариев.
- `src/main/kotlin/.../transport` - REST, SOAP, GraphQL и WebSocket маршруты.
- `src/main/kotlin/.../grpc` - gRPC-сервис и преобразования моделей.
- `src/main/proto` - protobuf-контракты.
- `src/test` - тесты маршрутов, контрактов и сценариев.

## Запуск

```bash
docker compose up --build
```

Порты по умолчанию:

- HTTP: `8080`.
- gRPC: `9090`.

## Проверка

```bash
gradle test
```

## Основные endpoints

- Healthcheck: `GET /health`.
- REST: `GET /api/rest/scenarios`, `GET /api/rest/scenarios/{scenario}`, `POST /api/rest/scenarios/{scenario}`.
- SOAP: `GET /api/soap/wsdl`, `POST /api/soap`.
- GraphQL HTTP: `POST /api/graphql`.
- GraphQL WebSocket: `/api/graphql/ws`.
- WebSocket: `/api/ws`.
- gRPC: сервис из `src/main/proto/experiment.proto` на порту `9090`.

## Методическое правило

Все транспортные ветки должны сохранять функциональную эквивалентность сценариев: одинаковую прикладную операцию, состав данных, модель ошибки, размер полезной нагрузки и правила завершения. Если меняется сценарий, схема ответа или правило расчета метрик, результаты старой и новой версии не объединяются.
